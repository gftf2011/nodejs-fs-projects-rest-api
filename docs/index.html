About

This project was built with intention to explore node.js internal modules, which means no Express. So, to fulfill this purpose, the most known modules were used! Among them are the 'fs' - (File System) module from node.js that is used to store our data, in the project it's not used any database management system like Postgres, Oracle OR MongoDB!

How it works?

To test the application check first if the Github Repo was cloned and it's running in your machine!

The application routes will use the localhost at port 8000!

Routes

HTTP Verb
GET

URL
http://localhost:8000/projects

Description
Endpoint responsible to read all projects.
Calling the endpoint will retrieve all projects in the api inside a vector.
HTTP Verb
GET

URL
http://localhost:8000/projects?id=number

Query String
Number represents the specific project's id passed in URL parameter!

Description
Endpoint responsible to retrieve a specific project given its id.
Calling the endpoint will retrieve a specific project object in JSON format.
HTTP Verb
POST

URL
http://localhost:8000/projects

Description
Endpoint responsible to create a project receiving in the request body the project's title.
Calling the endpoint will retrieve all projects in the api inside a vector.
HTTP Verb
POST

URL
http://localhost:8000/projects?id=number

Query String
Number represents the specific project's id passed in URL parameter!

Description
Endpoint responsible to create a task to a specific project given its id, also its passed through the request body the task for the project.
Calling the endpoint will retrieve all projects in the api inside a vector.
HTTP Verb
PUT

URL
http://localhost:8000/projects?id=number

Query String
Number represents the specific project's id passed in URL parameter!

Description
Endpoint responsible to update a specific project's title given its id, also its passed through the request body the new project's title.
Calling the endpoint will retrieve all projects in the api inside a vector
HTTP Verb
DELETE

URL
http://localhost:8000/projects?id=number

Query String
Number represents the specific project's id passed in URL parameter!

Description
Endpoint responsible to remove a specific project given its id.
Calling the endpoint will retrieve all projects in the api inside a vector
